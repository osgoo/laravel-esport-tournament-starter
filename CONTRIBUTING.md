# Contributing

Thanks for improving this starter.

## Local Checks

```bash
composer validate --no-check-lock
php artisan test
npm run build
```

## Scope

Contributions should stay within the public starter boundary: tournaments, teams, registration, admin approval, demo bracket/result flows, documentation, accessibility, and tests.

Do not add real-money wallet logic, payout workflows, payment provider integrations, tenant billing, production credentials, customer data, or private business copy.

## Pull Requests

- Keep changes small and explain the user-facing behavior.
- Include tests for behavior changes.
- Update documentation when setup or workflow changes.
